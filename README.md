# Order Bot — Telegram-бот для інтернет-магазину

AI-асистент на базі Claude, який консультує клієнтів, відповідає на FAQ та оформлює замовлення прямо в Telegram.

## Структура

```
order-bot/
├── bot.js           # головний код бота
├── catalog.js       # модуль читання каталогу
├── catalog.json     # список товарів (редагуйте під свій магазин)
├── package.json     # залежності
├── .env.example     # приклад змінних середовища
└── README.md
```

## Встановлення

1. Встановіть залежності:
   ```
   npm install
   ```

2. Створіть файл `.env` на основі `.env.example`:
   ```
   cp .env.example .env
   ```

3. Заповніть у `.env`:
   - `BOT_TOKEN` — токен вашого Telegram-бота (отримати у [@BotFather](https://t.me/BotFather))
   - `ANTHROPIC_API_KEY` — ваш ключ Anthropic API (console.anthropic.com)

4. Відредагуйте `catalog.json` — додайте свої реальні товари, ціни, розміри та наявність.

## Запуск

```
npm start
```

Бот почне слухати повідомлення в Telegram. Напишіть йому `/start`, щоб почати.

## Як це працює

- Кожен товар з `catalog.json` автоматично підставляється у системний промпт — бот ніколи не вигадує товари чи ціни.
- Історія діалогу зберігається в пам'яті процесу (по кожному chatId окремо) — для продакшену з кількома клієнтами варто перенести це у базу даних (наприклад, Redis чи SQLite), оскільки історія зараз втрачається при перезапуску бота.
- Модель: `claude-sonnet-5`.

## Що варто додати для продакшену

- Персистентне сховище для історії діалогів та замовлень (зараз усе — тільки в оперативній пам'яті)
- Логування замовлень (наприклад, у Google Sheets, CRM або базу даних) — зараз бот лише показує підсумок клієнту, але нікуди його не зберігає
- Сповіщення менеджеру про нове замовлення (наприклад, в окремий Telegram-чат)
- Ліміти на кількість запитів від одного користувача (rate limiting)
