# Order Bot — Telegram-бот для інтернет-магазину

AI-асистент на базі Claude, який консультує клієнтів, відповідає на FAQ,
оформлює замовлення прямо в Telegram (включно з Telegram Business) і
автоматично тримає актуальний каталог товарів через Google Таблицю.

## Структура

```
order-bot/
├── bot.js                          # головний код бота
├── manual-test.mjs                 # офлайн-тест логіки каталогу (npm test)
├── catalog.js, catalog.json        # СТАРІ файли, більше не використовуються ботом (можна видалити)
├── package.json                    # залежності
├── .env.example                    # приклад змінних середовища
├── examples/
│   └── catalog-template.csv        # приклад таблиці для імпорту в Google Sheets
├── docs/
│   └── GOOGLE_SHEETS_SETUP.md      # повний гайд по налаштуванню каталогу
└── src/
    ├── config/catalogConfig.js     # змінні середовища каталогу
    ├── utils/logger.js             # логування
    └── catalog/
        ├── googleClient.js         # автентифікація Google
        ├── googleSheetsService.js  # читання таблиці + retry
        ├── catalogMapper.js        # рядок таблиці -> об'єкт товару
        ├── catalogValidator.js     # перевірка коректності товару
        ├── catalogCache.js         # кеш у пам'яті + fallback при збоях
        ├── catalogSynchronizer.js  # оркестрація синхронізації
        ├── catalogSearcher.js      # fuzzy search + фільтри
        └── catalogRepository.js    # єдина точка входу для bot.js
```

## Встановлення

1. Встановіть залежності:
   ```
   npm install
   ```

2. Створіть файл `.env` на основі `.env.example`:
   ```
   cp .env.example .env
   ```

3. Заповніть у `.env` як мінімум:
   - `BOT_TOKEN` — токен вашого Telegram-бота (отримати у [@BotFather](https://t.me/BotFather))
   - `ANTHROPIC_API_KEY` — ваш ключ Anthropic API (console.anthropic.com)
   - `GOOGLE_SHEETS_SPREADSHEET_ID`, `GOOGLE_SERVICE_ACCOUNT_EMAIL`, `GOOGLE_PRIVATE_KEY` — див. **[docs/GOOGLE_SHEETS_SETUP.md](docs/GOOGLE_SHEETS_SETUP.md)**, повний покроковий гайд з нуля

4. Каталог товарів тепер ведеться в Google Таблиці, а не в `catalog.json`.
   Структура колонок і приклад для імпорту — там само, у
   `docs/GOOGLE_SHEETS_SETUP.md` та `examples/catalog-template.csv`.

## Перевірка перед запуском (без Google Sheets)

```
npm test
```

Прожене повний офлайн-тест логіки каталогу (мапінг, валідація, кеш,
пошук, фільтри) на прикладних даних — корисно, щоб переконатись, що
все встановилось коректно, перш ніж підключати реальну Google Таблицю.

## Запуск

```
npm start
```

Бот почне слухати повідомлення в Telegram (включно з Business-режимом,
якщо він у вас увімкнений). Напишіть йому `/start`, щоб почати.

## Як це працює

- Каталог товарів автоматично синхронізується з обраного джерела
  (Google Таблиця або CSV-файл — перемикається однією змінною
  `CATALOG_PROVIDER`) у фоні (за замовчуванням — раз на хвилину) і
  одразу підставляється у системний промпт AI. Бот ніколи не вигадує
  товари, ціни чи наявність — усе береться виключно з джерела даних.
- **Ціна в таблиці/файлі — закупівельна.** Бот автоматично рахує кінцеву
  ціну для клієнта, додаючи налаштовану націнку (`MARKUP_PERCENT` тощо
  в `.env`). Клієнт і AI ніколи не бачать закупівельну ціну — це
  структурно гарантовано, не просто домовленість (див.
  `docs/PRICING_AND_PROVIDERS.md`).
- Додати/змінити/приховати товар — просто відредагуйте Google Таблицю
  (для CSV — файл + передеплой). Деплой і перезапуск бота для Google
  Sheets НЕ потрібні.
- Власник може викликати `/refreshcatalog` в Telegram для миттєвого
  примусового оновлення (потрібно вказати свій `OWNER_TELEGRAM_ID` в `.env`).
- Якщо джерело даних тимчасово недоступне — бот продовжує працювати з
  останньою відомою версією каталогу, не падає.
- Історія діалогу зберігається в пам'яті процесу (по кожному chatId
  окремо) — для продакшену з кількома клієнтами варто перенести це у
  базу даних (наприклад, Redis чи SQLite), оскільки історія зараз
  втрачається при перезапуску бота.
- Модель: `claude-sonnet-5`.

Детальніше:
- Каталог, обробка помилок, пошук — **[docs/GOOGLE_SHEETS_SETUP.md](docs/GOOGLE_SHEETS_SETUP.md)**
- Націнка, округлення ціни, як додати новий провайдер (Shopify, Prom.ua тощо) — **[docs/PRICING_AND_PROVIDERS.md](docs/PRICING_AND_PROVIDERS.md)**

## Що варто додати для продакшену

- Персистентне сховище для історії діалогів та замовлень (зараз усе — тільки в оперативній пам'яті)
- Логування замовлень (наприклад, у окрему Google Таблицю, CRM або базу даних) — зараз бот лише показує підсумок клієнту, але нікуди його не зберігає
- Сповіщення менеджеру про нове замовлення (наприклад, в окремий Telegram-чат)
- Ліміти на кількість запитів від одного користувача (rate limiting)
- Якщо каталог виросте до тисяч товарів — переключити `bot.js` з повного `formatForPrompt()` на цільовий `search()`/`filter()` за ключовими словами з повідомлення клієнта (модулі вже готові, див. розділ 8 у гайді)
